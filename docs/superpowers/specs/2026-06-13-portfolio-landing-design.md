# Лендинг-портфолио Антона Малаховского — дизайн-спецификация

**Дата:** 2026-06-13
**Статус:** согласовано (готово к плану реализации)

## Цель

Рабочий лендинг — личное портфолио бренд-менеджера с упором на web-продукты.
Должен вызывать доверие и давать удобно пролистывать и изучать кейсы.

Референсы: juliencalot.com (главная, список, детальная), firstclass.partners
(прелоадер, поп-ап контактов). Метод контента — `.md`-файлы.

## Ключевые решения

| Вопрос | Решение |
|---|---|
| Стек | **Astro** (static output), Content Collections, View Transitions |
| Desktop/Mobile | **Responsive** (один кодовый набор, breakpoint ~768px) |
| Контент кейсов | **frontmatter (метаданные) + свободный markdown-тело** |
| Анимации | **«как референсы»**: анимированный прелоадер, View Transitions, scroll-reveal, видео-автоплей |
| Hero | **центрированная** композиция (вариант A) |
| Список кейсов | **scroll-snap, один кейс на экран** |
| Хостинг | GitHub Pages + GitHub Actions, задел под custom domain (CNAME) |

## Палитра и стиль

- Основа: белый/кремовый фон, чёрный текст. Минимализм в духе juliencalot.
- Цвет приходит из самих работ (Hero-фон `main.webp` — «Спортсмены» Малевича) и обложек кейсов.
- Hero-фон светлый → текст и имя тёмные, с лёгкой подложкой-градиентом для контраста поверх фигур.

## Архитектура

- `output: 'static'` — статические HTML/CSS/JS под GitHub Pages.
- **Content Collections** (`src/content/cases/`) — типобезопасная схема (zod) на метаданные + markdown.
- **Astro View Transitions** (`<ClientRouter />`) — переходы список↔кейс и управление восстановлением скролла.
- Анимации: CSS + лёгкая JS-обвязка (`IntersectionObserver` для появления, Web Animations API для прелоадера). Тяжёлые библиотеки (GSAP) — только при необходимости.
- Стили: один CSS-набор с переменными и media-запросами.

## Роуты

```
/                 → Preloader + Header + Hero + CaseList(snap) + Footer
/cases/[slug]     → детальная подстраница кейса (одна на каждый .md)
```

## Компоненты

- **Preloader** — оверлей: белый фон, «Loading», чёрный статус-бар, проценты 0→100%.
  Показывается только при первом заходе (initial load), не при внутренней навигации.
  Гибрид: анимируем прогресс, финал (100% → раскрытие) гейтим на загрузку
  критичных ассетов Hero (фон, аватар, шрифт).
- **Header** — sticky, белая полоса, кнопка «Контакты».
- **ContactsPopup** — десктоп: выезжает справа поверх Hero; мобайл: на весь экран.
  Ссылки:
  - email: `amunbelievable@gmail.com`
  - Telegram: `https://t.me/amunbelievable`
  - LinkedIn: `https://linkedin.com/in/anton-malakhovskiy-9452bb360`
- **Hero** — фон `main.webp` + круглый аватар + имя «Антон Малаховский» +
  «creative team lead» / «digital | web | strategy» + короткий текст «о себе».
  Центрированная композиция. Кнопка CONTACT.
- **CaseList** — секции со scroll-snap (`scroll-snap-type: y mandatory`), один кейс на экран.
  Строка кейса: название бренда слева · вижуал (img/video autoplay) по центру · годы справа.
  Мобайл: вертикаль (название → вижуал → годы). Клик → детальная.
- **CaseDetail** — десктоп: вижуал слева / описание справа; мобайл: вертикаль.
  Кнопка «Назад» внизу → возврат на главную к позиции этого кейса.
- **Footer** — кнопка «Наверх» (к Hero).

## Модель контента кейса

`src/content/cases/<slug>.md`:

```yaml
---
title: "CRASHER.MX"          # название бренда
years: "2024 — 2025"         # строка с годами
order: 1                      # позиция в списке (сортировка по возрастанию)
cover:
  type: "image"              # image | video
  src: "./media/crasher-cover.webp"
heading: "Бренд-менеджмент выхода на Мексику"  # подзаголовок детальной
---

Свободный markdown: заголовки, абзацы, картинки, видео.
Шаблон по умолчанию — Задача / Инсайт / Решение (не обязателен).
```

- Медиа лежат рядом с `.md`.
- Количество кейсов произвольное; список строится из коллекции, сортируется по `order`.
- Битый frontmatter ломает билд (типобезопасность через схему) — это желаемое поведение.

## Навигация, переходы и восстановление позиции

Требование: «Назад» возвращает в то место списка, где был пользователь.

- Клик по кейсу → переход на `/cases/[slug]` через View Transitions (мягкий fade / shared-element переход вижуала).
- Перед переходом сохраняем slug текущего кейса в `sessionStorage`.
- На детальной кнопка **«Назад»** → на `/` с указанием целевого кейса; при возврате
  scroll-snap **мгновенно** (без анимации перелистывания) устанавливается на нужный кейс.
- Браузерная кнопка «назад» обрабатывается тем же механизмом восстановления.

## Responsive

- Один кодовый набор, breakpoint ~768px.
- Desktop-раскладки — слайды 3–5 презентации, mobile — слайды 6–8.
- Видео: `muted`, `loop`, `playsinline`, `autoplay`.

## Деплой

- GitHub Actions: push в `main` → `astro build` → publish в GitHub Pages.
- `CNAME` для будущего custom domain.
- `site` / `base` в `astro.config` настроить под итоговый адрес.

## Тестирование

- Сборка проходит без ошибок; контент валидируется схемой коллекции.
- Ручной чек-лист:
  1. Прелоадер (0→100%, только initial load) → раскрытие.
  2. Hero отображается корректно поверх `main.webp`, текст читается.
  3. Снап-скролл по кейсам (десктоп — колесо, мобайл — свайп).
  4. Клик в кейс → плавный переход на детальную.
  5. «Назад» возвращает ровно на тот кейс, где был пользователь.
  6. «Наверх» из футера → к Hero.
  7. Контакты: десктоп — справа поверх Hero; мобайл — фуллскрин; все 3 ссылки рабочие.
  8. Desktop и mobile раскладки на ключевых ширинах.
- Lighthouse — перформанс и доступность.
